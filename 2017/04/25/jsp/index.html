转发和重定向的区别
转发的工作方式：
用户->JSP页面请求->服务器中请求响应->结果反馈给客户端

重定向：
客户端->请求服务器响应->响应->再次请求->新地址响应给客户端

使用会话对象session实现
一个会话就是浏览器与服务器之间的一次通话
会话可以在多次请求中保存和使用数据

session id

public void setAttribute(String name,Object value);

public Object getAttribute(String name);
用法：String userName=(String)session.getAttribute(“userName”);

每个session都有一个唯一的sessionid

浏览器向服务器发送请求->sessionid FCA000…
响应->（携带sessionid）

查看session ID
session.getId();

session的数据是在服务器端的，服务器保存的会话数据量会越来越大
，从而导致性能问题

若没有清理机制，会导致性能问题或服务器崩溃
程序主动清除session数据
服务器主动清除长时间没有再次发出请求的session
